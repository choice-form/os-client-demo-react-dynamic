这是一个基于React的动态组件加载的online survey clinet程序,它来自于模板仓库[os-client-demo-react-dynamic](https://github.com/choice-form/os-client-demo-react-dynamic).

这是一个动态模板解决方案,请查看[编码规则](./doc/dynamic-rules.md);


已知:
+ 集成了tailwind
+ 实现了如错误提示,预览工具,进度条,计时器等所有全局组件.
+ 实现了开始页面,结束页面,奖励页面,验证节点这几个节点类型入口组件,其他节点类型的入口组件没有实现.

建议:

有了动态加载功能以后,意味着每个节点类型的入口组件的写法都会更加自由奔放,不会像以前会把大量心思考虑在很多部分抽出共通组件,在这种情况下,我们后续编码时,尽量遵循节点类型入口组件之间不要依赖共通的功能组件,只依赖共通的样式组件的规则.跨入口组件调用的共通的功能组件对现在这种编程方式来说会是一种绑架,一种阻碍.
